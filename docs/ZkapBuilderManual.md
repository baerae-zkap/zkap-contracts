# ZkapBuilder Usage Manual

## 1. Overview

ZkapBuilder is a builder class for creating and managing ZKAP (Zero Knowledge Account Protocol) wallets. This document explains how to use ZkapBuilder, AccountKeyBuilder, and ZkapCreator.

## 2. Full Process

The ZKAP wallet creation process is as follows:

1. **Key Generation**

   - Generate master key (ZkOAuthRS256)
   - Generate transaction key (WebAuthn)
   - Encode keys using AccountKeyBuilder

2. **Wallet Creation**

   - Prepare data for wallet creation using ZkapCreator
   - Generate UserOp

3. **Transaction Execution**

   - Submit the UserOp generated by ZkapCreator as a handleOps execution transaction

The key update process for an existing ZKAP is as follows:

1. **Key Generation**

   - Generate transaction key (WebAuthn)
   - Encode keys using AccountKeyBuilder

2. **Key Update**

   - Generate key update data using ZkapBuilder

3. **Transaction Execution**

   - Submit the UserOp generated by ZkapBuilder as a handleOps execution transaction

## 3. Key Generation (AccountKeyBuilder)

AccountKeyBuilder is responsible for generating and encoding various types of keys used in ZKAP wallets.

### 3.1 Master Key Generation (ZkOAuthRS256)

#### Method 1: Step-by-step creation

```typescript
const accountKeyBuilder = new AccountKeyBuilder();

const threshold = 1;
const keyInfoList: ZkOAuthRS256KeyInfo[] = [
  {
    weight: 1,
    userSpecificVk: userVk, // per-user verification key
  },
];

const masterKey = accountKeyBuilder.setZkOAuthRS256Key(threshold, keyInfoList);
```

#### Method 2: Create and finalize in one step

```typescript
const threshold = 1;
const keys: KeyInfo[] = [
  {
    keyType: PrimitiveAccountKeyTypes.keyZkOAuthRS256,
    weight: 1,
    keyData: {
      userSpecificVk: userVk,
    },
  },
];

const keyBuilder = new AccountKeyBuilder(threshold, keys);
const masterKey = keyBuilder.getEncodedCompositeKey();
```

### 3.2 Transaction Key Generation (WebAuthnKey)

#### Method 1: Step-by-step creation

```typescript
const threshold = 1;
const keyInfoList: WebAuthnKeyInfo[] = [
  {
    weight: 1,
    credentialPubkey: "0x1234567890abcdef",
    credentialId: "0x1234567890abcdef",
    rpIdHash: "0x1234567890abcdef",
    origin: "https://example.com",
  },
];

const keyBuilder = new AccountKeyBuilder();
const txKey = keyBuilder.setWebAuthnKey(threshold, keyInfoList);
```

#### Method 2: Create and finalize in one step

```typescript
const threshold = 1;
const keys: KeyInfo[] = [
  {
    keyType: PrimitiveAccountKeyTypes.keyWebAuthn,
    weight: 1,
    keyData: {
      credentialPubkey: "0x1234567890abcdef",
      credentialId: "0x1234567890abcdef",
      rpIdHash: "0x1234567890abcdef",
      origin: "https://example.com",
    },
  },
];

const keyBuilder = new AccountKeyBuilder(threshold, keys);
const txKey = keyBuilder.getEncodedCompositeKey();
```

## 4. Wallet Creation (ZkapCreator)

ZkapCreator is responsible for creating ZKAP wallets.

### 4.1 Initialization

```typescript
const zkapCreator = new ZkapCreator({
  chainId: 8216,
  entryPoint: entryPoint.target.toString(),
  zkapFactory: zkapAccountFactory.target.toString(),
  keyFactory: compositeFactory.target.toString(),
  enUrl,
  txKeySigner: new AddressKeySigner([ownerWallet.privateKey]),
  salt: ethers.sha256(ethers.toUtf8Bytes(`${userId}${account_num}`)),
  encodedMasterKey: masterKey,
  encodedTxKey: txKey,
});
```

(See section 7. Signers for txKeySigner assignment)

### 4.2 Send wallet creation tx

```typescript
// Send generated UserOp (the user's ZKAP must have sufficient funds for fees)
await zkapCreator.completeUserOp();
const tx = await entryPoint.handleOps(
  [zkapCreator.getPackedUserOp()],
  ownerWallet.address
);
```

## 5. Key Update (ZkapBuilder)

ZkapBuilder is responsible for updating keys of an existing ZKAP wallet.

### 5.1 Initialization

```typescript
const zkapBuilder = new ZkapBuilder({
  chainId: 8216,
  entryPoint: entryPoint.target.toString(),
  enUrl,
  txKeySigner: new AddressKeySigner([privateKey]),
  masterKeySigner: new ZkOAuthRS256KeySigner(serverUrl, enUrl, accountAddress, [
    idTokenGenerator,
  ]),
});
```

(See section 7. Signers for txKeySigner and masterKeySigner assignment)

### 5.2 Transaction Key Update

```typescript
// 1. Generate new transaction key
const threshold = 1;
const keyInfoList: WebAuthnKeyInfo[] = [
  {
    weight: 1,
    credentialPubkey: "0x1234567890abcdef",
    credentialId: "0x1234567890abcdef",
    rpIdHash: "0x1234567890abcdef",
    origin: "https://example.com",
  },
];

const keyBuilder = new AccountKeyBuilder();
const newtxKey = keyBuilder.setWebAuthnKey(threshold, keyInfoList);

// 2. Build and send UserOp
zkapBuilder.setSender(zkapAddress).setUpdateTxKeyCallData(newTxKey);

await zkapBuilder.completeUserOp();

const tx = await entryPoint.handleOps(
  [zkapBuilder.getPackedUserOp()],
  ownerWallet.address
);
```

## 6. Transaction Execution

```typescript
await zkapBuilder.completeUserOp();

const tx = await entryPoint.handleOps(
  [zkapBuilder.getPackedUserOp()],
  ownerWallet.address
);
```

## 7. Signers

ZKAP wallets support various types of signers. Each signer is responsible for generating a signature appropriate for a specific key type.

### 7.1 ZkOAuthRS256KeySigner

ZkOAuthRS256KeySigner is a signer that generates OAuth RS256 signatures. It integrates with OAuth services such as Google, Apple, and Kakao to generate signatures.

```typescript
const zkOAuthRS256Signer = new ZkOAuthRS256KeySigner(
  proofServerUrl, // ZK proof server URL
  enUrl, // Ethereum node URL
  zkapAddress, // ZKAP wallet address
  [idTokenGenerator] // array of ID token generator functions (insert per-provider social login idToken generator functions here)
);

// Initialize
await zkOAuthRS256Signer.init();

// Generate signature
const signatures = await zkOAuthRS256Signer.signUserOpHash(userOpHash);
```

Features:

- Supports OAuth RS256 signatures
- Generates signatures by integrating with the ZK proof server
- Supports multi-signatures (multiple ID token generator functions can be passed)
- Supports OAuth services such as Google, Apple, and Kakao

### 7.2 PasskeySigner

PasskeySigner is a signer that generates signatures using WebAuthn/Passkey. It generates signatures using the user's biometric authentication or security key.

```typescript
const passkeySigner = new PasskeySigner(
  credentialId, // WebAuthn credential ID
  verifyWithPasskey // Passkey verification function
);

// Generate signature
const signatures = await passkeySigner.signUserOpHash(userOpHash);
```

Features:

- Supports WebAuthn/Passkey signatures
- Supports biometric authentication and security keys
- Signature is generated on the client side
- Supports single signature only

## 8. Notes

1. Sufficient ETH or paymaster fees are required when creating a wallet.
2. Transaction key updates must be signed with the master key.
3. Regular transactions must be signed with the transaction key.

## 9. Error Handling

- `Required fields are missing`: Required fields have not been set
- `UserOpSigner is not set`: Signer has not been set
- `Invalid key type`: An invalid key type was used
- `Threshold is greater than the sum of weights`: Threshold exceeds the sum of key weights
