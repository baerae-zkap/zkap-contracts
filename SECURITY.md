# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, please email: **security@baerae.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 1 week
- **Fix timeline:** Depends on severity, typically within 2 weeks for critical issues

## Scope

This policy applies to the smart contracts in this repository. For vulnerabilities in the ZKAP SDK (`@baerae/zkap-aa`), please report to the same email address.

## Severity Classification

- **Critical:** Loss of funds, unauthorized access to wallets, signature bypass
- **High:** Denial of service, griefing attacks, gas manipulation
- **Medium:** Information disclosure, non-standard behavior
- **Low:** Gas optimization, code quality issues

## Acknowledgments

We appreciate responsible disclosure and will acknowledge reporters in our security advisories (with permission).
